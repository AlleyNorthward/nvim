# 当前配置到此为止了
@author 巷北  
@time 2025.11.21  

嗯~下面有什么就直接说什么了, 我也不进行拆分了. 一开始参考了某个小册, 进行配置. 不过那个小册是2022年的了, 前面的还好, 可是到了lsp后, 问题就多了起来. 我自己了解了一下, 发现, 当前最流行的插件管理器是lazyvim, 而小册中带我用的是packer. 这段时间也一直思考, 分析, 纠错, 但回过头来一看, 似乎一直在原地踏步. 看了看自己的配置, 还是两三天前的配置. 记得9月份, 刚了解到vim时, 那时候操作也不太熟练, vim配置全是ai配置的, 确实十分困难, 因为是vim相关的脚本语言, 学习起来也不是很容易. 所以就一直问ai, 一直问ai, 最后搞的有些不耐烦了, 弄了两三天吧, 虽然可以用, 但最后还是回到vscode了. 不过在vscode中, 也是一直联系vim相关的操作, 逐渐地熟练了起来. 过了两个月后, 也就是最近这三四天吧, 又突然转向了vim, 只不过这次深入的是neovim.(9月份当时确实什么都不明白, 先弄的gvim, 搞的差不多了, 发现用起来一卡一卡的, 才了解到neovim, 测试了下确实非常快速, 所以两个月后直接操作neovim了)找了些资料, 发现很少, 可是也知道了, 当前neovim官方推荐使用lua语言配置vim.lua我了解啊, 有了具体语言来配置, 方便了很多. 一开始还好, 跟着小册配置, 后面摸索按键映射, 了解了很多, 也自己配置了很多映射, 并写了些逻辑, 符合我自己的个人习惯. 可随后, 发现lsp配置起来非常麻烦, 于是又回到了两个月之前那样, 一直问ai, 一直问ai. 现在回过头来反思一下, 还是想先迅速配置出来, 然后再深入了解. 可是迅速配置谈何容易呢?????  
所以还是打算, 放慢脚步.ai有些滥用了.确实, ai可以帮助我快速配置, 可当它无法满足我的需求后, 一切又会从头做起. 假如, 两个月前, 我慢慢了解vim底层的实现原理, 而不是靠ai去配置, 不急于求成, 那么如今我是什么样呢?站在今天, 假想两个月后, 我慢慢地从neovim底层了解, 那时候, 我又是什么样呢?????  
ai给我带来了什么?什么情况下应该使用ai,什么情况下又不该使用ai,而是慢慢地学习呢?  
很早之前其实也说过, 使用ai是为了加深理解的, 快速明白是什么的, 而不是问ai, 这么怎么做, 给出具体代码的.以vim为例, 我应该去问, 具体的语句是什么含义, 有没有其他类似的语句, 具体是什么意思, 不是去问, 我应该怎么配置. 是fast understand, 不是fast finish. 是快速理解, 不是快速结束.可能, 一时半会我也配置不好neovim(主要还是指lsp, 自动补全等), 但我会深入了解neovim相关接口, 明白插件与其的交互. 因为后续大部分时间都会泡在代码之中, 有一个好的武器, 还是很有必要的.
